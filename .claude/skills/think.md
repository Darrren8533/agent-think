# /think — 装载领域思维框架

## Trigger
当用户输入 `/think <domain>` 时触发此 skill。
支持叠加：`/think financial engineering`（同时装载多个思维）

## Available Thinks
- `financial` — 金融思维（ROI、风险、机会成本）
- `engineering` — 工程思维（量化、瓶颈、简单优先）
- `product` — 产品思维（用户行为、最小验证、问题优先）

## Steps

1. 读取用户指定的 think 文件（`thinks/<domain>.md`）

2. 宣告激活：
   告诉用户你已装载哪个思维框架，用一句话说明这个框架的核心视角。

3. 同时激活两层：
   - **视角层**：用该领域的核心本能分析问题
   - **质疑层**：在给出分析之前，先检查这个框架是否适合当前问题

4. 如果用户叠加了多个思维，说明不同视角之间的张力在哪里
   （例如：金融视角说"砍掉这个功能省钱"，产品视角说"这个功能是用户核心需求"）

5. 在对话过程中，当发现当前思维框架不适合时，主动提示切换或叠加。

## Output Format

装载确认：
```
已装载 [Financial Think]
核心视角：任何决策先算钱，风险优先于收益，关注机会成本。
质疑层已激活：我会在分析之前先检查金融框架是否适合你的问题。

现在告诉我你想分析什么。
```

## Important
- think 不是万能的。如果用户的问题不适合当前框架，直接说出来
- 质疑层不是反驳用户，是帮用户看到框架的边界
- 叠加多个 think 时，不同视角矛盾是正常的——矛盾本身就是洞察
