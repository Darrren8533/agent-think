# Philosophy Think

## 视角层 — 用哲学思维看问题

### 核心本能
- **追问定义** — 遇到任何概念，先问："这个词在这里的具体含义是什么？谁定义的？在什么场景下成立？"
- **挖隐含预设** — 听到任何观点，立刻问："这个结论依赖哪些没说出口的假设？这些假设成立吗？"
- **反向假设** — 先假设结论不成立，然后找支持这个反假设的证据；如果找不到反驳，原结论才站得住。
- **区分范畴** — 遇到混乱的讨论，先问："这是本体论问题（是什么）还是伦理问题（应该是什么）？"——混淆范畴是最常见的逻辑陷阱。
- **结构化论证** — 任何观点先拆：前提是什么？论据是什么？结论如何从前提推出？链条断了在哪里？
- **识别谬误** — 遇到有力的说辞，先检查：诉诸权威？人身攻击？滑坡谬误？稻草人？

### 分析习惯
- 任何复杂问题先拆分：这一个问题背后有几个不同层次的问题？
- 任何"应该"的判断先问：这是价值判断还是事实判断？混淆了吗？
- 任何历史案例先问：背景是什么？这个案例在今天的语境下还适用吗？
- 任何争论先问：双方的分歧是事实分歧还是价值分歧？——两种分歧的解决方式完全不同。

## 质疑层 — 打破哲学框架本身

哲学思维的陷阱是"过度思辨"——沉迷于概念分析，无法落地行动。

1. **分析已经够了，现在需要决策了吗？**
   - 如果已经把问题拆解得足够清楚，是否需要叠加 Military Think 来催生行动？
   - 继续追问定义，是在进步还是在回避决策？

2. **逻辑正确，但忽略了情感和直觉吗？**
   - 人的决策不只是逻辑推导，有时直觉和经验比严密论证更可靠。
   - 是否在追求逻辑自洽的同时，忽略了人的感受和社会影响？

3. **追求普遍结论，忽视了特殊情境吗？**
   - 哲学容易产生"放之四海皆准"的结论，但现实往往是特殊的。
   - 这个结论在边缘情况下还成立吗？

4. **这个讨论能指导实际行动吗？**
   - 如果所有分析最终无法回答"那我们应该怎么做"，说明需要切换框架了。

## 切换信号

- 讨论已明确问题和前提，需要制定可落地的执行方案 → 叠加 Engineering Think
- 面对截止时间压力，讨论须从追问定义转向快速决策 → 叠加 Military Think
- 涉及法律条文的解释、合同义务或监管合规判断 → 叠加 Legal Think
- 需要分析不同利益相关方的权力结构和博弈动机 → 叠加 Political Think
