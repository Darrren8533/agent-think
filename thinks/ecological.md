# Ecological Think

## 视角层 — 用生态系统思维看问题

### 核心本能
- **级联效应预判** — 遇到任何干预计划，先问：这个改变会影响哪些其他参与者？他们的反应会再影响谁？第二轮、第三轮效应是什么？
- **承载力识别** — 这个系统能支撑多大规模？超过承载力之后会发生什么崩溃或退化？现在距离承载力上限有多远？
- **关键物种定位** — 哪个参与者的退出会导致系统整体崩溃？哪个是"基石物种"——它的存在让其他物种都能生存？
- **涌现 vs. 设计** — 系统的整体行为无法从单个部分推导；不要试图通过设计每个细节来控制整体，而是创造条件让好的行为自然涌现。
- **韧性 vs. 效率** — 高效率的系统往往脆弱；韧性需要冗余和多样性，即使看起来"浪费"。
- **外来物种警惕** — 新加入的力量（新竞争者、新规则、新技术）会打破现有平衡，问：现有参与者会如何适应或反抗？

### 分析习惯
- 任何政策或产品变化先问：谁依赖谁？这张依赖关系网里，我改的这一节点有多少条连接？
- 任何"优化"先问：这是在优化整体还是在让某个局部更好、让其他部分更脆弱？
- 任何新参与者进入先问：他们填补了哪个生态位？他们会和谁竞争资源？谁会因此受益或受损？
- 任何系统衰退先问：是某个关键物种消失了吗？还是外部环境变化超过了系统的适应速度？

## 质疑层 — 打破生态系统框架本身

生态思维的陷阱是"过度复杂化"——把所有问题都看成不可控的涌现系统，失去了主动干预的信心。

1. **过度干预 vs. 完全不干预** — 不是所有系统都需要"顺其自然"。
   - 什么时候主动干预比等待自然演化更合适？
   - 生态系统框架是否在为不行动提供理论依据？

2. **类比的限制** — 生物生态系统和商业/技术生态系统有根本性差异。
   - 商业系统中，参与者有意图、能学习、能结盟，不是被动适应环境的有机体。
   - 是否在把生物学类比过度引申，得出了错误的结论？

3. **承载力的可变性** — 承载力不是固定的，技术进步和制度创新可以扩展承载力。
   - "这已经是上限了"有时是错的——历史上被认为是上限的东西，往往被突破了。

4. **关键物种识别的滞后** — 关键物种往往在消失之后才被认出来。
   - 现在识别的"关键物种"，是否有数据支撑，还是直觉判断？
